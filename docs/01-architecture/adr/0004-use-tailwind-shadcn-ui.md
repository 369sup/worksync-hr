# ADR-0004: 使用 Tailwind CSS + shadcn/ui

## 狀態
Accepted

## 日期
2026-06-29

## 背景
- 專案需要快速建立一致的管理後台 UI，且不希望導入高維護成本的自製設計系統。

## 決策
- 採用 Tailwind CSS 作為樣式基礎。
- 採用 shadcn/ui 作為主要元件組合方式。

## 原因
- 元件組裝速度快。
- 樣式可維持一致性。
- 可降低自製元件維護成本。
- 與 Next.js / TypeScript 整合自然。
- 易於建立表單與管理介面。

## 取捨
- 優點：設計一致、開發效率高、客製彈性夠。
- 限制：仍需治理元件變體與使用規則。

## 後續影響
- 架構上 UI 層需集中管理元件使用規則。
- 程式碼需優先重用既有 shadcn/ui 元件。
- Firebase 不得因 UI 便利性而直接暴露到 Client Component。
- 文件需同步維護元件、表單與頁面規範。
